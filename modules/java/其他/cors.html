CORS ​

概念 ​

CORS (Cross-Origin Resource Sharing) 是一种浏览器安全机制，用于控制跨来源的 HTTP 请求。 当客户端 JavaScript 代码通过 Ajax 或其他方式将请求发往服务器，如果请求和脚本源不在相同的域名或端口上，浏览器会拒绝该请求，以防止被恶意站点攻击。 通过在服务器上设置 Access-Control-Allow-Origin 等请求头，可以允许特定域名和端口的请求。

spring环境下配置跨域 ​

通过继承 WebMvcConfigurer 类重写 addCorsMappings 方法实现跨域 ​

java
@Configuration
public class CorsConfig implements WebMvcConfigurer {

    @Override
    public void addCorsMappings(CorsRegistry registry) {
        // 设置允许跨域的路径
        registry.addMapping("/**")
                // 设置允许跨域请求的域名
                .allowedOriginPatterns("*")
                // 是否允许cookie
                .allowCredentials(true)
                // 设置允许的请求方式
                .allowedMethods("*")
                // 设置允许的header属性
                .allowedHeaders("*")
                // 跨域允许时间
                .maxAge(3600);

    }

简单使用可以. 但是当有配置自定义拦截器的时候, 仍然会报跨域问题. 因为请求会先进入拦截器而不是Mapping映射中, 所以返回的头信息中没有配置的跨域信息. 浏览器就会报跨域异常. 例如: 配置了一个登录拦截器, 众所周知, 发送跨域请求的时候, 浏览器会事先发送一个options请求, 用于判断服务是否支持跨域访问. 此时, 这个options请求将会被拦截器拦截, 由于其没有携带token信息, 因此不会放行, 不会被配置的Mapping映射添加相应的响应头. 浏览器以该返回信息进行判断服务器不支持跨域, 抛出跨域异常.

使用CorsFilter实现 ​

java
@Configuration
public class CorsConfig implements WebMvcConfigurer {

    @Autowired
    private LoginInterceptor loginInterceptor;

    @Bean
    public CorsFilter corsFilter() {
        //1. 添加 CORS配置信息
        CorsConfiguration config = new CorsConfiguration();
        //放行哪些原始域
        config.addAllowedOriginPattern("*");
        //是否支持用户凭证
        config.setAllowCredentials(true);
        //放行哪些请求方式
        config.addAllowedMethod("*");
        //放行哪些原始请求头部信息
        config.addAllowedHeader("*");
        //暴露哪些头部信息
        config.addExposedHeader("*");
        //2. 添加映射路径
        UrlBasedCorsConfigurationSource corsConfigurationSource = new UrlBasedCorsConfigurationSource();
        corsConfigurationSource.registerCorsConfiguration("/**", config);
        //3. 返回新的CorsFilter
        return new CorsFilter(corsConfigurationSource);
    }

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(loginInterceptor)
                .addPathPatterns("/**")
                // 放行 swagger
                .excludePathPatterns("/doc.html")
                .excludePathPatterns("/swagger-resources")
                .excludePathPatterns("/webjars/**")
                // 放行登录请求
                .excludePathPatterns("/user/register/**")
                .excludePathPatterns("/user/userLogin/**");
    }
}
