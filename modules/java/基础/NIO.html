NIO ​

概念 ​

new io 又称为 non-blocking io

核心组成 ​

  • buffer 缓冲区
  • channel 管道
  • selector 选择器

其中, channel不进行数据处理, 仅负责数据运输, 由 buffer 负责与数据打交道

channel ​

常用类型 ​

  • FileChannel
    从文件读取数据

  • DatagramChannel

    Datagram-数据报

    通过 UDP 在网络上读取和写入数据

  • SocketChannel
    通过 TCP 在网络上读取和写入数据

Buffer ​

结构说明 ​

  • position
    类似于读/写指针, 表示当前读(写)到什么位置
  • limit
    在写模式下表示最多能写入多少数据, 此时和 capacity 相同。在读模式下表示最多能读多少数据, 此时和缓存中的实际数据大小相同
  • capacity
    在读/写模式下都是固定的, 即分配的缓冲大小(容量)

常用方法 ​

flip ​

从写模式变成读模式

limit 变为 position 当前值, 然后 position 置 0, 清除 mark(mark 修改为 -1), 表示变为 读操作, 并从缓存的头部开始读取, 最多读取到之前写入的数据长度位置

注意：读写转换时, buffer.flip() 一定得有, 否则就是每次都从缓存最后开始读取(错误读取)

clear ​

将 position 置 0, 将 limit 置为 capacity 的大小, 清除 mark (mark 修改为 -1) 表示变为 写操作, 准备下一次写入

compact ​

类似于 clear 作用, 但是保留未阅读数据 将未读数据放入缓存头部, 将 position 指针指向该部分数据的末尾, limit 仍然指向 capacity 位置, 然后准备继续写入

rewind ​

将 position 置 0, 清除 mark, 它的作用在于为提取 Buffer 的有效数据做准备

mark ​

设置标记, 后续使用 reset 方法, 使 position 回到当前标记的位置

reset ​

回到之前标记的位置

transferFrom ​

将数据从源通道传输到当前通道

java
// 将数据从 fromChannel 移动到 toChannel
// 从 position 位置开始, 传输 count 字节

transferTo ​

将数据从当前通道传送到目标通道

java
fromChannel.transferTo(position, count, toChannel)

简单示例 ​

java
// NonReadableChannelException 表示尝试读取一个没有进行写入过的channel
String fileName = "word";
FileInputStream fis = new FileInputStream(fileName);
FileChannel channel = fis.getChannel();

// // 另一种创建方式

ByteBuffer buffer = ByteBuffer.allocate(4 * 1024);
int len;
while((len=channel.read(buffer))!=-1){
    // 翻转buffer  position 置 0
    buffer.flip();
    // 读取数据
    System.out.println(new String(buffer.array(), 0, len));
    // 清空buffer
    buffer.clear();
}

词频统计 ​

java
package coding.buffer;

import java.io.*;
import java.nio.MappedByteBuffer;
import java.nio.channels.FileChannel;
import java.nio.charset.StandardCharsets;
import java.util.*;
import java.util.concurrent.*;

/**
 * 多线程词频统计
 *
 * @author zuijianren
 * @date 2021/7/7 16:40
 */
public class WordCount2 {
    final ForkJoinPool pool = ForkJoinPool.commonPool();

    private static HashMap<String, Integer> countByString(String str) {
        HashMap<String, Integer> map = new HashMap();
        StringTokenizer stringTokenizer = new StringTokenizer(str);
        while (stringTokenizer.hasMoreTokens()) {
            String word = stringTokenizer.nextToken();
            incKey(map, word, 1);
        }
        return map;
    }

    private static void incKey(HashMap<String, Integer> map, String key, Integer n) {
        if (map.containsKey(key)) {
            map.put(key, map.get(key) + n);
        } else {
            map.put(key, n);
        }
    }

    public void run(String fileName, long chunkSize) throws ExecutionException, InterruptedException {
        File file = new File(fileName);
        long fileLength = file.length();

        HashMap<String, Integer> totalMap = new HashMap<>();
        ArrayList<Future<HashMap<String, Integer>>> tasks = new ArrayList<>();

        long position = 0;
        long start = System.currentTimeMillis();
        while (position < fileLength) {
            long next = Math.min(position + chunkSize, fileLength);
            CountTask task = new CountTask(fileName, position, next);
            position = next;
            Future<HashMap<String, Integer>> future = pool.submit(task);
            tasks.add(future);
        }
        for (Future<HashMap<String, Integer>> future : tasks) {
            HashMap<String, Integer> map = future.get();
            for (Map.Entry<String, Integer> entry : map.entrySet()) {
                incKey(totalMap, entry.getKey(), entry.getValue());
            }
        }
        System.out.println(System.currentTimeMillis() - start + "ms");
        System.out.println(totalMap.get("ababa"));
        System.out.println(totalMap.size());
    }


    // 执行结果: 
    // 177ms
    // 307
    // 3133
    public static void main(String[] args) throws ExecutionException, InterruptedException {
        WordCount2 wordCount2 = new WordCount2();
        wordCount2.run("word", 1024 * 1024);
    }

    // 内部任务类
    class CountTask implements Callable<HashMap<String, Integer>> {
        private final long start;
        private final long end;
        private final String fileName;

        CountTask(String fileName, long start, long end) {
            this.start = start;
            this.end = end;
            this.fileName = fileName;
        }

        @Override
        public HashMap<String, Integer> call() throws Exception {
            FileChannel channel = new RandomAccessFile(this.fileName, "rw").getChannel();
            // 由：  device -> kernel space -> user space -> thread
            // 变为： device -> kernel space -> thread
            // 即 将 部分内核空间，短暂的映射，作为 用户空间 使用
            MappedByteBuffer buffer = channel.map(
                    FileChannel.MapMode.READ_ONLY,
                    start,
                    end - start
            );
            String s = StandardCharsets.US_ASCII.decode(buffer).toString();
            return countByString(s);
        }
    }
}

IO 和 NIO 的区别 ​

IONIO
面向流面向缓冲
阻塞 IO非阻塞IO
无选择器

面向流与面向缓冲 ​

IO: 面向流, 意味着每次从流中读取一个或多个字节, 直至读取所有字节, 它们没有被缓存在任何地方. 此外, 无法前后移动流中的数据. 如果需要前后移动流中读取的数据, 需要先将它缓存到一个缓存区 NIO: 面向缓冲, 数据读取到一个它稍后处理的缓冲区, 需要时, 可以在缓冲区中前后移动. 这就增加了处理过程中的灵活性. 但是, 还需要检查该缓冲区中包含所有需要处理的数据. 而且, 确保需要更多数据读入缓冲区时, 不要覆盖缓冲区中尚未处理的数据.

阻塞与非阻塞IO ​

IO: 各种流是阻塞的. 当一个线程调用 read 或 write时, 该线程被阻塞, 直到数据处理完成, 在此期间, 数据不能处理任何事 NIO: 非阻塞模式, 一个线程请求写入一些数据到某通道, 不需要等待它完全写入, 这个线程同时可以去做别的事情. 线程通常将非阻塞IO的空闲时间用于在其它通道上执行IO操作,所以一个单独的线程现在可以管理多个输入和输出通道

选择器 ​

允许一个单独的线程来监视多个输入通道, 你可以注册多个通道使用一个选择器, 然后使用一个单独的线程来“选择”通道: 这些通道里已经有可以处理的输入, 或者选择已准备写入的通道. 这种选择机制, 使得一个单独的线程很容易来管理多个通道

/**
* 校验上级目录是否存在, 没有则进行创建
*
* @param file 要创建的文件
*/
private static void validateDirExist(File file) {
    while (!file.getParentFile().exists()) {
        validateDirExist(file.getParentFile());
        boolean mkdirs = file.getParentFile().mkdirs();
        if (!mkdirs) {
            throw new RuntimeException("创建文件夹失败");
        }
    }
}

获取指定路径下所有类的方法 ​

注意: 此方法无法获取 jar 包中的类 当前替代方法为: 使用 hutool 包中的 ClassScanner.scanPackage(packageName) 方法获取

java
// 指定获取的路径   也可使用传参的方式指定
    private static final String ENUM_BASE_PACKAGE = "com.zkxn.common.enums";

    // 类名列表  =>  存放解析结果
    // 可以通过此对象  日志打印解析结果
    private static final List<String> CLASS_NAME_LIST = new ArrayList<>();

    // 扫描指定文件夹  获取  其下的所有类
    private void doScan(String basePackage) throws IOException {
        String packageFilePath = basePackage.replaceAll("\\.", "/");
        // 获取类加载器
        ClassLoader classLoader = Thread.currentThread().getContextClassLoader();
        assert classLoader != null;
        // 获取 resources
        Enumeration<URL> resources = classLoader.getResources(packageFilePath);
        // 遍历获取到的url
        while (resources.hasMoreElements()) {
            URL url = resources.nextElement();
            String path = url.getPath();
            File file = new File(path);
            File[] files = file.listFiles();
            for (File childFile : files) {
                if (childFile.isDirectory()) {
                    doScan(basePackage + "." + childFile.getName());
                } else if (childFile.getName().endsWith(".class")) { // 对 .class 文件进行解析
                    String className = basePackage + "." + childFile.getName().replaceAll(".class", "");
                    CLASS_NAME_LIST.add(className);
                }
            }
        }
    }

中文路径乱码原因及解决方法 ​

  • 原因: 中文路径乱码 是因为 我们此处获取路径的对象是 url 对象, url对象会将路径中的 中文字符进行转码, 因此 对获取到的路径需要添加一次解码处理
